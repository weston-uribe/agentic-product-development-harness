# V0.3.0 — Release contract

**Tag status:** Not yet tagged

**npm status:** Not yet published

**Release type:** GitHub **source release** (annotated tag + curated release notes) plus public npm CLI package **`p-dev@0.3.0`**.

## TL;DR

V0.3.0 is the guided-onboarding and distribution release. Product managers can start with `npx --yes p-dev@0.3.0` without cloning the source repository. The seven-step Configure GUI automates service credentials, Linear workspace setup, Vercel webhook bridge, target repos, cloud secrets, and guarded workflow install PR finalization.

## Who this is for

- Product managers and solo operators who want guided harness onboarding
- Contributors who develop the harness from the source repository
- Teams already on Linear + GitHub + GitHub Actions + Vercel + Cursor

## Primary install command

```bash
npx --yes p-dev@0.3.0
```

Requires Node.js 22+. macOS is the validated packaged platform; use `--no-open` elsewhere.

Full guide: [`docs/p-dev.md`](../p-dev.md)

## Architecture / workspace model

| Boundary | Role |

## Setup flow

1. Connect services (optional packaged harness workspace provisioning)
2. Set up Linear workspace
3. Set up Vercel webhook bridge
4. Choose target repo(s)
5. Check local readiness
6. Connect cloud secrets
7. Install target repo workflow (guarded auto-finalization for system-owned setup PRs)

Remote mutations require explicit confirmation. Ordinary product PRs remain governed by Linear status gates.

## Security model

- Secrets in local workspace, GitHub Actions secrets, and Vercel env vars — never in the published package
- Confirmation-gated remote writes with fingerprint validation
- Output redaction before logs and artifacts
- Published tarball excludes credentials and operator state

Details: [`docs/security.md`](../security.md)

## Known limitations

- Cursor-only agent provider
- Linear / GitHub / GitHub Actions / Vercel stack
- macOS validated for packaged browser auto-launch
- No full isolated npm issue lifecycle validated
- No automatic private harness workspace upgrade/sync
- Public template compatibility dependency
- Not production SaaS or provider-agnostic
- Manual eval rubrics where automation is not implemented
