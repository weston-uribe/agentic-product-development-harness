# Release contract: v0.3.1 (preparation only)

**Status:** Release preparation documentation only. **Do not** publish npm, create tags, or create GitHub releases from this document alone.

## Summary

`p-dev-harness@0.3.1` is a backward-compatible patch release that binds each published package version to an **immutable embedded workspace snapshot** and provisions new private workspaces from that snapshot without calling the public template repository at runtime.

## What ships

- Embedded `workspace-snapshot/manifest.json` + `workspace-snapshot/files/**` in the npm tarball
- Package-owned GitHub provisioning (`createUserRepository` + git blob/tree/commit APIs)
- Managed marker v3 with `createdFromPackageSnapshot` provenance
- Legacy reconnect for valid `p-dev-harness@0.3.0` managed workspaces
- Conditional release-impact guidance in planner/implementation contracts

## What does not ship

- Automatic upgrade of existing managed workspaces to the 0.3.1 snapshot
- Changes to `weston-uribe/p-dev-harness-template` default branch or tags
- Autonomous npm publication, git tags, or GitHub releases

## Human-gated validation checklist

Run from a **clean checkout** at the intended `RELEASE_SHA`:

```bash
npm ci
npm run build
npm test
P_DEV_SNAPSHOT_SOURCE_REF="$RELEASE_SHA" npm run package:p-dev:prepare
npm run package:p-dev:pack
```

Record evidence:

| Field | Command / location |
|-------|-------------------|
| `RELEASE_SHA` | merge commit on `main` |
| Snapshot `sourceCommit` | `packages/p-dev/workspace-snapshot/manifest.json` |
| `snapshotContentId` | same manifest |
| `snapshotSha256` | same manifest |
| Tarball path | `packages/p-dev/p-dev-harness-0.3.1.tgz` |
| Tarball SHA-256 | `shasum -a 256 packages/p-dev/p-dev-harness-0.3.1.tgz` |

Tarball must include:

- `package/workspace-snapshot/manifest.json`
- Curated `package/workspace-snapshot/files/**`
- Existing launcher/GUI assets

Tarball must exclude:

- `.env.local`, `.harness/config.local.json`, caches, `*.tgz`, generated package build dirs

Provisioning smoke (human-gated, real account):

1. Install the local tarball in a fresh operator workspace
2. Run Configure Step 1 fresh provisioning against a private test destination
3. Verify destination repo marker v3 provenance matches embedded manifest
4. Verify snapshot commit parent tree equals `gitRootTreeSha1`
5. Record duration, blob count, retry count, and any rate-limit backoff

## Legacy template containment

`weston-uribe/p-dev-harness-template` remains a **frozen** compatibility artifact for `p-dev-harness@0.3.0`. Do not resync template `main` for 0.3.1+.

## Rollback

- Unpublished code: revert merge commit on `main`
- Published npm: follow npm deprecation/unpublish policy only under explicit operator approval; prefer forward-fix `0.3.2` for defects
